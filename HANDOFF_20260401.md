# RouteFuel Handoff

## Current State

The iOS app is now working against the RouteFuel backend contract with a map-first UI and a compact route-planning flow.

Implemented changes:

1. Added bundled app config in `RouteFuel/RouteFuel/Config.plist`, following the same pattern used by Meeting Hero.
2. `EndpointConfiguration.live()` now reads `AIServiceURL` and `AuthToken` from `Config.plist`, with environment override support still present.
3. Default live config now points to `http://devaiservices.traland.com/api/routefuel` and sends `X-RouteFuel-Key`.
4. Request handling was split so route creation uses a shorter timeout and fuel-stop search uses a long timeout suitable for the current backend.
5. Fuel-stop response parsing was relaxed to accept JSON numbers delivered as either integers or floating-point values.
6. ATS was enabled for the current HTTP dev backend through the app plist.
7. The search flow was reworked into a compact planner:
   - `From`
   - `To`
   - one shared active search field
   - calculate route
8. Origin selection now supports either current location or a searched custom start point.
9. Results view now shows map actions for both Apple Maps and Google Maps.
10. Google Maps now falls back to the web URL if the native app is unavailable.
11. Apple Maps launch now includes all three points in order: start, selected fuel stop, destination.
12. Added `.gitignore` to exclude macOS junk, Xcode user state, and the local `temp/` folder.

## UI State

Current route flow:

1. Select `From`:
   - current location
   - or a searched origin
2. Select `To` from destination search results.
3. Calculate route.
4. Review recommended stops in the bottom sheet.
5. Open the selected stop in:
   - Apple Maps
   - Google Maps, only when available

The recent mobile layout cleanup is in place:

1. Search screen is compact and no longer over-stacked.
2. Results screen is map-dominant with a proportional bottom sheet.
3. Fuel-stop cards and route overlays are readable on phone-sized screens.

## Backend / Integration Notes

Current config values:

1. Base URL: `http://devaiservices.traland.com/api/routefuel`
2. Header: `X-RouteFuel-Key`
3. Timeout for fuel-stop search: `250` seconds

Observed backend behavior:

1. `POST /v1/routes` is working.
2. `POST /v1/fuel-stops/search` now returns valid data for the tested route.
3. Fuel-stop search is still slow relative to the intended contract, so the client currently tolerates that with a longer timeout instead of an async job model.

## Known Technical Notes

1. Apple Maps launch behavior is now correct, but the implementation still uses deprecated iOS 26 MapKit APIs around `MKPlacemark` and `MKMapItem(placemark:)`.
2. `xcodebuild build -project RouteFuel/RouteFuel.xcodeproj -scheme RouteFuel -destination 'generic/platform=iOS Simulator' -derivedDataPath /tmp/RouteFuelDerivedData` succeeds.
3. Simulator-backed tests are still not runnable in this environment because `CoreSimulatorService` is unavailable.
